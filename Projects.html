Content Websites Platform
Duration Jan 2016 - Current
Team Size 20
Url(s) login.cwp.gov.sg
Role Sitecore lead, Solution Architect
Environment AWS Cloud, Node.js, PowerShell, C#, Asp.net, .Net Framework 4.7,Sql Server 2014, Sitecore 9.0, 8.2, 8.1, 8.2 & 7.2, Sitefinity 10.2, 10.1, 10.0, 9.2 & 9.1, SharePoint 2013 & 2016, WordPress, SWIIIT, Sophos, Nginx
Description
This project is about providing a hosting environment for many government websites in public cloud with five of CMS. We choose AWS as hosting provider. The main objective of this project is automation as this project will host 400+ websites. Main objective of this project are fast provisioning and auto scaling. For auto provisioning, we set up an engine where we can specify CMS and it will create website with specified CMS and domain. All the things created in this project should be high availability mode too. We customized AWS for all 5 CMS and achieved auto provisioning through engine. For auto scaling, the requirement is we have to spawn new VM for websites when current resource is not enough for traffic and terminate a VM when current resource is more than enough. But we always must maintain at least 2 VM. We achieved it by utilizing AWS auto scaling concept. We are also required to harden security. We achieved it by using AWS security groups, subnets and IAM roles.

  • Involvement in setting up AWS configurations and codes for Sitecore CMS.
  • Involvement in fixing Pen test and other security issues for Sitecore.

  • Solutioning and Developing for new features into CWP for CMS
  • Responsible for advising Sitecore CMS clients with solutions and alternatives with challenges they face to on board from current hosting platform to CWP
  • On call support when there are incidents on Sitecore CMS applications

Websites Content Platform System(WCMS)
Duration April 2014 - Dec 2019
Team Size 10
Role Log Server set up and Sitecore Lead
Environment Nxlog, Logstash, Apache Solr, Kibana Winscp SFTP, C#, Asp.net,.Net Framework 4.5, entity framework, sql server, Sitecore
Description
This is a bulk tender for hosting government websites. Customers can host Sitecore and Sitefinity websites on this platform

Centralized log server concept: We setup centralized log server with nxlog -> logstash -> Apache Solr -> kibana. The concept of this logging is as follows. There will be two kind of zones, MAZ and HAZ. HAZ will host web, app and db tiers. MAZ will host web server. Public traffic will be routed as Internet Load Balancer -> MAZ web server -> HAZ app server. Intranet traffic will be routed as Intranet Load Balancer -> HAZ web server -> HAZ app server. For log server we need to forward logs (IIS logs, event logs, performance logs and application logs) from all servers to a centralized log server. And need to create graphs and dashboards to visualize data from all servers.

Sitecore and Sitefinity: Infrastructure is designed for Sitecore CMS and Sitefinity with all kind of government IM8 security policies. For both CMS High Availability or Normal mode designs are designed, so customer can choose design according to their budget.
Responsibilities
Development Phase

  • Centralized Log server set up
  • Involved in Sitecore projects architecture planning
  • Involved in Development, bug fixing and security tests fixing for Sitecore POC Website

Maintenance Phase

  • Advice and Debug on the problems faced on Sitecore and Sitefinity
  • Solutioning for new versions or any customizations required according to user requirements
  • On call support when there are incidents on CMS applications
